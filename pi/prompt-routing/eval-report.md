## 5. Security Assessment

**Risk**: `pickle.load()` executes arbitrary Python bytecode during deserialization.

**Mitigations implemented**:

1. **SHA256 pre-verification** — `evaluate.py` and any inference code must verify `model.pkl.sha256` before loading. Mismatch → hard exit, no deserialization.
2. **Local generation** — `model.pkl` is generated by `train.py` on the local machine. Never load a `model.pkl` received over a network or from an untrusted source.
3. **Sidecar co-location** — `model.pkl.sha256` must be stored adjacent to `model.pkl` and checked before every load.

**Residual risk**: If an attacker can modify both `model.pkl` and `model.pkl.sha256` simultaneously (i.e., has write access to the artifact directory), the SHA256 check is bypassed. Mitigation: restrict filesystem permissions on the `prompt-routing/` directory in production.

## 6. Model Architecture Decision Log

The board consensus specified `LinearSVC + CalibratedClassifierCV`. During implementation, `cv='prefit'` (which would allow a single SVM at inference) was discovered to have been **removed in scikit-learn 1.8.0**. Benchmarking all available options:

| Configuration | Mean Inference | Decision |
|---------------|---------------|---------|
| `CalibratedClassifierCV(cv=5)` | ~3700 us | Rejected: 5x SVM ensemble exceeds budget |
| `CalibratedClassifierCV(ensemble=False)` | ~1557 us | Rejected: still exceeds 1ms budget |
| `LinearSVC` direct | ~490 us | **Selected**: meets all hard constraints |

The production pipeline uses `LinearSVC` directly. `CalibratedClassifierCV(cv=5)` is retained in the **grid search phase only** for stable cross-validation score computation during C selection. The production model does not need probability calibration — hard routing requires only class labels.
