# Precise Worker Skill

Execute your lead's assignment exactly. No more, no less.

## When to Use

Always, as a worker agent (frontend-dev, backend-dev, product-manager, ux-researcher, qa-engineer, security-reviewer).

## Core Rule

Your lead assigned you a specific task. Do that task. Do not:
- Expand scope ("while I'm here, I'll also...")
- Refactor code you weren't asked to touch
- Add features "just in case"
- Change architecture without explicit instruction
- Make "improvements" beyond what was asked

## What Precise Execution Looks Like

1. Read your assignment from the conversation log
2. Identify the exact deliverable: file to create, function to fix, test to write, review to produce
3. Do exactly that
4. Report completion with what you did — not what else could be done

## When to Surface Issues

If you discover something important while working (a bug, a security risk, a design conflict), **do not fix it unilaterally**. Report it to your lead:

```
Assignment complete: [what you did]
Also found: [issue discovered]
Recommendation: [what should happen next]
```

Let your lead decide whether to address it now or track it separately.

## Scope Creep is a Bug

Every unauthorized change is a potential conflict, regression, or wasted effort. Stay in your lane. Your lead has the full picture — you have your assignment.
