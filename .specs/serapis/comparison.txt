  vs SOPS

  ┌──────────────────────┬──────────────────────────────────────────────────────────────────────────┬────────────────────────────────────────────────────────────────────────────────┐
  │         Area         │                                   SOPS                                   │                                    Serapis                                     │
  ├──────────────────────┼──────────────────────────────────────────────────────────────────────────┼────────────────────────────────────────────────────────────────────────────────┤
  │ Zero-knowledge       │ Server/git host never sees plaintext — encryption happens client-side    │ Server sees plaintext during encryption. Server compromise = everything leaked │
  ├──────────────────────┼──────────────────────────────────────────────────────────────────────────┼────────────────────────────────────────────────────────────────────────────────┤
  │ No infrastructure    │ Just a CLI + git repo. Nothing to host, nothing to back up (besides git) │ Requires running a server, Docker host, backups, KEK management                │
  ├──────────────────────┼──────────────────────────────────────────────────────────────────────────┼────────────────────────────────────────────────────────────────────────────────┤
  │ Offline              │ Works completely offline, it's just files                                │ Requires network access to the vault                                           │
  ├──────────────────────┼──────────────────────────────────────────────────────────────────────────┼────────────────────────────────────────────────────────────────────────────────┤
  │ Battle-tested crypto │ Uses age/AWS KMS/GCP KMS — mature, audited implementations               │ Custom Go crypto (HKDF, AES-GCM wrapping) — correct by design but unaudited    │
  ├──────────────────────┼──────────────────────────────────────────────────────────────────────────┼────────────────────────────────────────────────────────────────────────────────┤
  │ Git integration      │ Encrypted files live in the repo, full diff/blame/history natively       │ Separate system from git, history lives in the vault's audit log               │
  └──────────────────────┴──────────────────────────────────────────────────────────────────────────┴────────────────────────────────────────────────────────────────────────────────┘

  Serapis's only real advantage over SOPS: push/pull UX and central sync without git. If you're comfortable with git-based workflows, SOPS is strictly more secure.

  The Honest Summary

  ┌─────────────────────────┬──────────┬─────────────────────────────────────────────────────────────────────────────────────────────────┐
  │        Weakness         │ Severity │                                           Mitigation                                            │
  ├─────────────────────────┼──────────┼─────────────────────────────────────────────────────────────────────────────────────────────────┤
  │ Server sees plaintext   │ High     │ Accepted tradeoff. Could add client-side encryption later, but that's basically rebuilding SOPS │
  ├─────────────────────────┼──────────┼─────────────────────────────────────────────────────────────────────────────────────────────────┤
  │ Single point of failure │ High     │ Backups + KEK backup procedure. Local .env files are the fallback                               │
  ├─────────────────────────┼──────────┼─────────────────────────────────────────────────────────────────────────────────────────────────┤
  │ Unaudited crypto        │ Medium   │ Use only stdlib/x/crypto packages, stick to well-known patterns, but it's still your code       │
  ├─────────────────────────┼──────────┼─────────────────────────────────────────────────────────────────────────────────────────────────┤
  │ No CI/CD integration    │ Medium   │ Scriptable CLI (envault pull in CI), but no native integrations                                 │
  ├─────────────────────────┼──────────┼─────────────────────────────────────────────────────────────────────────────────────────────────┤
  │ No dynamic secrets      │ Low      │ Out of scope — Serapis is a static vault, not a secret engine                                   │
  ├─────────────────────────┼──────────┼─────────────────────────────────────────────────────────────────────────────────────────────────┤
  │ No GUI                  │ Low      │ By design — CLI-first for developers                                                            │
  ├─────────────────────────┼──────────┼─────────────────────────────────────────────────────────────────────────────────────────────────┤
  │ Bus factor of 1         │ High     │ It's your project. If you stop maintaining it, it stops getting security patches                │
  └─────────────────────────┴──────────┴─────────────────────────────────────────────────────────────────────────────────────────────────┘

  The recurring theme: every established tool has either more security (SOPS, 1Password) or more features (Vault, Doppler) than Serapis. What Serapis offers is a specific UX (shebang + push/pull + SSH keys) at zero cost with zero
   third-party trust, at the expense of being less secure than client-side encryption and less featureful than managed platforms.
