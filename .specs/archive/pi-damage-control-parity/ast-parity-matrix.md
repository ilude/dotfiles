# Damage-Control Parity Matrix

Scope: comparison of `claude/hooks/damage-control/` with Pi runtime files `pi/extensions/`, `pi/lib/`, `pi/tests/`, and `pi/damage-control-rules.yaml` after the Bash AST port.

| Area | Claude coverage | Pi coverage | Status | Evidence | Remaining gap |
|---|---|---|---|---|---|
| Bash AST analysis | `ast_analyzer.py` uses Tree-sitter Bash as a veto-only second pass. | `pi/extensions/damage-control/ast-analyzer.ts` uses `web-tree-sitter` plus `tree-sitter-bash` wasm as a veto-only second pass from `evaluateDangerousCommand`. | Closed for priority AST cases. | `pi/tests/damage-control-ast.test.ts` covers shell `-c`, subshell, pipeline, eval, dynamic eval/source, variable expansion, safe fast path, and veto-only behavior. | Not a byte-for-byte port; unsupported grammar edge cases remain possible. |
| Rule coverage | Claude canonical `patterns.yaml` has broad Bash patterns and AST config. | Pi loads Claude policy when available and has Pi fallback rules with `astAnalysis` config. | Mostly closed for loaded command/path policy. | `pi/extensions/damage-control-rules.ts`; `pi/damage-control-rules.yaml`. | Exfil, taint, sequence, and semantic git behaviors remain separately tracked. |
| Path protections | Claude supports zero access, exclusions, read-only, no-delete, write-confirm, content scan. | Pi maps these sections to read/write/edit/bash/pwsh handlers where Pi has equivalent tool surfaces. | Mostly closed for Pi tool surfaces. | `pi/extensions/damage-control.ts`; existing `damage-control.test.ts`. | Post-tool output scanning is not implemented. |
| Content scanning | Claude has content scan and injection patterns. | Pi checks configured content paths and injection patterns on write/edit. | Partial. | `contentNeedsScan` and `containsInjectionPattern` in `pi/extensions/damage-control-engine.ts`. | Secret-output and broader exfil content analysis remain open. |
| Context relaxations | Claude has context-aware relaxations and read-only search handling. | Pi has read-only search handling for common search/inspection and CLI read-only subcommands plus supported `--dry-run` relaxation for Bash pattern checks. | Mostly closed for command evaluation. | `isReadOnlySearchCommand`, `hasValidDryRun`, `stripDockerEnvFileArgs` in `pi/extensions/damage-control-engine.ts`; `pi/tests/damage-control-parity-gaps.test.ts`. | Named Claude context profiles are not imported as runtime profile config; Pi uses deterministic built-ins. |
| Sequence detection | Claude has `sequence-patterns.yaml` behavior. | Pi tracks session-local read/glob/bash summaries and catches the configured sensitive-file/network, credential-search/network, AWS creds/S3, SSH key/network, DB dump/cloud, env enumeration exfil, tfstate/network, and kubeconfig/network patterns. | Closed for equivalent sequence families. | `pi/extensions/damage-control-state.ts`; `pi/extensions/damage-control.ts`; `pi/tests/damage-control-parity-gaps.test.ts`. | Pi state is in-memory per session rather than persisted to Claude's JSON state file. |
| Audit/debug behavior | Claude writes JSON audit logs. | Pi writes eval telemetry, permission registry decisions, status, debug log, and warns/records on secret-looking tool output. | Native parity. | `pi/lib/damage-control-eval.ts`, `pi/extensions/damage-control-debug.ts`, `recordBlock` and `tool_result` handling in `pi/extensions/damage-control.ts`. | Log schema intentionally remains Pi-native, not Claude-compatible JSONL. |
| Ask/block semantics | Claude ask exits with prompt JSON; block exits nonzero. | Pi ask uses UI confirmation and denies when unavailable or declined; AST can only escalate. | Closed for covered Pi tools. | `evaluateDangerousCommand`; `damage-control-ast.test.ts`; `damage-control.test.ts`. | None for covered ask/block contract. |
| Tests | Claude has pytest suites including `test_ast_analyzer.py`. | Pi has Vitest coverage in `damage-control.test.ts` and `damage-control-ast.test.ts`. | Closed for ported priority AST tests. | Validation commands listed in this goal. | Full Claude pytest fixture parity runner remains open. |

## Explicit unclosed gaps

1. Named Claude context profiles are not imported as profile data; Pi implements the safe readonly/dry-run relaxations directly.
2. Pi sequence and taint tracking are session-local in memory, not persisted to Claude's JSON state files.
3. Pi audit logs remain native telemetry/debug records, not the same JSONL schema as Claude hooks.
