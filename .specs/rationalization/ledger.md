# Test decision ledger

## Basis and reconciliation

This ledger uses declaration-level IDs. A parametrized declaration appears once as `test_id[...]`; it is not repeated for each collected parameter case. Direct source verification corrected the prior audit from **89 STRICT / 106 BROAD declarations** to **90 STRICT / 107 BROAD declarations**. The prior count omitted `test/test_config_patterns.py::test_install_idempotent`, which reads tracked source and asserts the literal `cmp -s`. The corrected 107 BROAD declarations comprise 80 Python declarations and 27 Pi Vitest declarations. All STRICT declarations are included in BROAD; 17 are BROAD-only.

The exact full-suite collection counts are **1222 Pytest tests** and **1334 Vitest tests**. Those are collected test-case counts across their suites, not static-contract candidate declaration counts. Pytest parametrization expands the 80 Python candidate declarations to 213 exact candidate IDs: 198 IDs from the 65 `test_config_patterns.py` declarations plus 15 singleton IDs. The full 1222 Pytest collection also includes tests outside this candidate definition. The 27 Vitest static-content candidates are declarations selected from the full 1334-test collection.

One additional runtime-output wording test is included because T6 names it explicitly: `workflow-commands-pure.test.ts` builds a runtime prompt rather than reading tracked prose, but its assertion freezes wording rather than validated classifier behavior. It is outside the 90/107 static-content reconciliation, bringing the execution ledger to 108 rows.

Inventory evidence established that `reportsTo`, `routingUse`, and `leads` were not parsed, while `roleType` was parsed but not launch-consumed. T4 removed all four fields and replaced taxonomy checks with consumed-field parse-to-launch coverage.

## Exclusion boundary

Excluded tests execute behavior, parse configuration and evaluate its runtime effect, or assert runtime-produced output. Examples in the evidence include prompt-builder output, skill-review artifact generation, Fable hook output, route-normalization fixture behavior, provider parity, damage-control rule enforcement, sandboxed memory-scan execution, installer subprocess tests, and behavioral capture or migration tests. These behavior/runtime-output exclusions are not candidates merely because their assertions contain strings.
